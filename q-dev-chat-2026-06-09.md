Based on the FRD, here are the required frameworks organized by layer:

## Frontend
- **Next.js 14** (App Router) — React-based SSR/SSG framework
- **Tailwind CSS** — Utility-first styling
- **shadcn/ui** — Component library
- **i18next** — Localization (Amharic + English per Section 23)

## Backend
- **Node.js + NestJS** — Modular, enterprise-grade REST/GraphQL API
- **tRPC** *(optional)* — Type-safe API layer between frontend/backend

## Authentication & Security
- **Keycloak** — RBAC/OIDC identity provider (Section 7, 9)
- **JWT + OAuth 2.0** — Token-based auth

## Database & Storage
- **PostgreSQL** — Primary relational DB (double-entry ledger, SIS)
- **Redis** — Caching, session management
- **MinIO** — S3-compatible object storage (on-premise, data sovereignty compliance per Section 25)

## Real-Time / Live Classroom
- **LiveKit** — WebRTC-based live classroom (explicitly mentioned in Section 1.2.4)

## AI & RAG
- **LangChain.js + Ollama** — Local RAG/LLM pipeline (Section 18, explicitly "Local RAG/LLM")
- **pgvector** — Vector embeddings in PostgreSQL

## Finance
- **custom double-entry ledger** built on PostgreSQL (Section 12)
- **Chapa / Telebirr SDK** — Ethiopian e-payment integration (NBE Directive, Section 1.4)

## Assessment / Proctoring
- **TensorFlow.js** — Browser-based face detection for proctoring (Section 15)

## Infrastructure
- **Docker + Docker Compose** — Containerization
- **Nginx** — Reverse proxy
- **PostgreSQL WAL / Immutable Audit Log** — Compliance logging (Section 19)

## Key Constraints Driving Choices
- All services must be **self-hostable on-premise** in Ethiopia (data sovereignty)
- Offline/low-bandwidth fallback required (Finote Selam context, Section 1.2.3)
- No third-party cloud storage (MinIO over S3)
- Local LLM only (Ollama over OpenAI)
